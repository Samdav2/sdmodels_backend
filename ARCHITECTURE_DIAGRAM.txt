                              DATA FLOW EXAMPLE
═══════════════════════════════════════════════════════════════════════════════

  User uploads 3D model:
  
  1. Client → POST /api/v1/models (with file)
  2. API Layer → Validates request, extracts user from JWT
  3. Service Layer → Business logic (check limits, process file)
  4. Repository Layer → Save to database
  5. Storage Service → Upload file to S3/R2
  6. Email Service → Send confirmation email
  7. Response → Return model details to client

  ✓ Repository Pattern      - Data access abstraction
  ✓ Service Pattern         - Business logic encapsulation

  ✓ Factory Pattern         - Object creation
  ✓ Strategy Pattern        - Payment methods
  ✓ Observer Pattern        - Event notifications
