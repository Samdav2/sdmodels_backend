┌─────────────────────────────────────────────────────────────────────────────┐
│                         8. SECURITY FEATURES                                │
└─────────────────────────────────────────────────────────────────────────────┘

    ✅ Atomic Transactions
       └─ All balance operations happen in single DB transaction
       └─ Either all succeed or all fail (no partial updates)

    ✅ Balance Constraints
       └─ Database CHECK constraints prevent negative balances
       └─ available_balance >= 0
       └─ held_balance >= 0

    ✅ Authorization Checks
       └─ Only bounty poster can approve/cancel
       └─ Only claimed artist can submit work

    ✅ Escrow Protection
       └─ Funds locked until work approved
       └─ Cannot be withdrawn or used for other bounties

    ✅ Complete Audit Trail
       └─ Every transaction logged with timestamp
       └─ Balance before/after recorded
       └─ Reference to bounty/milestone

    ✅ Reference Tracking
       └─ Each transaction links to bounty_id
       └─ Can trace all funds for any bounty

╚══════════════════════════════════════════════════════════════════════════════╝

Create Bounty:  POST /api/v1/bounties
                → Checks balance
                → Holds funds in escrow
                → Creates bounty

Check Balance:  GET /api/v1/wallet/balance
                → Returns available, held, total

Deposit Funds:  POST /api/v1/wallet/deposit
                → Adds to available balance

Approve Work:   POST /api/v1/bounties/{id}/approve
                → Releases payment to artist
                → Deducts 7.5% platform fee

Cancel Bounty:  DELETE /api/v1/bounties/{id}
                → Refunds held funds
                → Returns to available balance

View History:   GET /api/v1/wallet/transactions
                → Shows all transactions
                → Filterable by type/status
